# Contributing

Thank you for your interest in contributing to PSGroup 1C Code Quality Tools.

## How to contribute

You can contribute by:

- Improving documentation
- Adding practical 1C code quality rules
- Adding examples of good and bad 1C code
- Suggesting static analysis checks
- Improving AI-assisted code review prompts
- Reporting issues and improvement ideas

## Contribution rules

- Keep examples practical and easy to understand
- Follow 1C:Enterprise coding standards
- Use clear naming
- Do not include client confidential data
- Do not include proprietary 1C configuration files without permission
- Explain why each quality rule is useful
- Add examples where possible

## Pull requests

Before submitting a pull request:

1. Describe the problem you are solving.
2. Explain what has changed.
3. Add examples if applicable.
4. Make sure no confidential data is included.
5. Make sure the content is useful for the 1C developer community.
